# =========================================================

# Federated IoT Botnet Detection System

# Setup & Execution Guide

# =========================================================

## Requirements

* Python 3.11 (Recommended)
* Windows 10 / 11
* Git (Optional)

---

## 1. Clone or Extract the Project

Clone the repository:

```bash
git clone <repository_url>
```

or simply extract the ZIP archive.

---

## 2. Open the Project Directory

Open PowerShell or Command Prompt.

Navigate to the project folder.

Example:

```bash
cd "E:\Sastra\Summer Project\Code"
```

---

## 3. Create a Virtual Environment

```bash
py -m venv venv
```

Activate it:

```bash
venv\Scripts\activate
```

The terminal should display:

```text
(venv)
```

---

## 4. Install Dependencies

```bash
pip install -r requirements.txt
```

---

## 5. Verify Project Structure

The following folders should exist:

```text
Code/
│
├── app.py
├── predict.py
├── requirements.txt
├── README.md
├── INSTRUCTIONS.txt
│
├── dataset/
├── models/
│   ├── autoencoder_model.h5
│   ├── federated_autoencoder.keras
│   ├── scaler.pkl
│   └── thresholds.json
│
├── scripts/
├── static/
├── templates/
└── uploads/
```

---

## 6. Launch the Web Application

Run:

```bash
py app.py
```

or

```bash
python app.py
```

The application will start on:

```text
http://127.0.0.1:5000
```

---

## 7. Open the Dashboard

Open your browser and visit:

```text
http://127.0.0.1:5000
```

---

## 8. Run Threat Analysis

1. Select a model:

   * Centralized Autoencoder
   * Federated Autoencoder

2. Upload a supported CSV file.

3. Click **Analyze**.

The dashboard displays:

* Prediction (Benign / Attack)
* Reconstruction Error
* Detection Threshold
* Threat Score
* Feature Contribution Analysis
* Confusion Matrix
* Model Comparison

---

## 9. Re-generate Evaluation Charts (Optional)

If evaluation charts need to be regenerated:

```bash
cd scripts

py evaluate_models.py
```

This regenerates:

```text
static/charts/

centralized_confusion.png
federated_confusion.png
model_comparison.png
feature_importance.png
model_metrics.json
centralized_metrics.json
federated_metrics.json
```

---

## 10. Retrain the Models (Optional)

### Centralized Autoencoder

```bash
py scripts/autoencoder.py
```

### Federated Autoencoder

```bash
py scripts/run_fl.py
```

---

## 11. Regenerate Detection Thresholds

After retraining the models:

```bash
py scripts/find_threshold.py

py scripts/find_threshold_federated.py
```

These scripts regenerate models/thresholds.json, which is used by both live prediction and the evaluation pipeline.

---

## 12. Regenerate the Shared Scaler (Only if Training Data Changes)

If the benign training dataset is modified or replaced:

```bash
py scripts/create_scaler.py
```

This creates:

```text
models/scaler.pkl
```

The same scaler is reused during:

* Centralized training
* Federated training
* Evaluation
* Live prediction

to ensure consistent preprocessing across the entire project.

---

## Troubleshooting

### ModuleNotFoundError

Activate the virtual environment:

```bash
venv\Scripts\activate
```

Then reinstall dependencies:

```bash
pip install -r requirements.txt
```

---

### Dataset Not Found

Ensure the following folder exists:

```text
dataset/
```

---

### Model Not Found

Ensure the following files exist:

```text
models/

autoencoder_model.h5
federated_autoencoder.keras
scaler.pkl
thresholds.json
```

---

### Port Already in Use

Run Flask on another port:

```bash
flask --app app run --port 5001
```

---

## Notes

* Python 3.11 is recommended.
* Evaluation charts are generated once and reused by the dashboard.
* If the models are retrained:

  1. Regenerate thresholds.
  2. Re-run `evaluate_models.py`.
* The application uses a single shared Min-Max scaler (`scaler.pkl`) for training, evaluation, and inference to ensure reproducible preprocessing.
